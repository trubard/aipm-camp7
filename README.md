# 일상 연구

> 주변을 사진으로 찍어 보내면 AI 팀장이 1~5분짜리 초소형 생활 과업으로 만들어 주고,
> 직접 해보며 *다른 고립청년의 일상 회복을 위한* 과업을 발굴·검증하는 회복형 생활 서비스의 MVP.

고립·은둔 청년을 위한 **B2G 회복형 생활 SaaS**입니다.

---

## 프로젝트 개요

- **목적** — 고립·은둔 청년이 초소형 생활 과업 수행을 통해 일상의 리듬을 회복하도록 돕는다.
- **배경** — 거듭된 실패로 생겨난 무기력과 '회복이 필요한 대상으로 여겨지는 데 대한 저항' 때문에, 회복 지원이 닿지 못하는 고립·은둔 청년이 늘고 있다.
- **목표** — 타인의 회복을 도울 일상 과제 발굴 연구팀의 기여자로서, *사진 제출 → AI 팀장의 과업 생성 → 수행 피드백*으로 이어지는 '발굴-검증' 단일 루프를, 가입 직후 30분 내 첫 성공 경험(Aha)과 함께 주 3일 이상 반복되게 만든다.

## 핵심 컨셉 — helper-frame

사용자를 '회복받아야 할 당사자'가 아니라 **'다른 고립청년을 위한 과업을 발굴·검증하는 연구팀 기여자'**로 세웁니다.
'회복 대상화'에서 오는 수치심·저항을 걷어내고, 기여·소속감으로 실행 동기를 만드는 것이 설계의 핵심입니다.
(본인의 일상 회복은 이 과정의 결과이지, 화면에 내거는 목적이 아닙니다.)

## 핵심 경험 — 발굴-검증 루프

1. **발굴** — 사용자가 주변에서 과업으로 만들 만한 장면(어지러진 방·설거지·분리수거 등)을 사진으로 제출
2. **과업 생성** — AI 팀장이 사진을 분석해 1·3·5분짜리 초소형 후보 과업을 제시
3. **수행·피드백** — 사용자가 직접 해보고 경험(난이도 등)을 피드백
4. **완료 보상** — 검증 완료 서머리(역량 인정 + 기여 적립 + 자라는 새싹)로 무압박 보상

> 위기(자해·자살 등) 신호가 감지되면 과업을 즉시 멈추고 정서적 지지와 상담 전화(109)를 우선 안내합니다.

## 기술 스택

| 영역 | 사용 기술 |
|---|---|
| 프론트엔드 | 바닐라 JS + HTML/CSS (프레임워크·빌드 없음, 단일 파일) |
| 백엔드 | Node.js (ESM) 서버리스 함수 |
| AI / LLM | Anthropic Claude `claude-haiku-4-5` · `@anthropic-ai/sdk` (구조화 출력 + vision, 팀장→상담가 2단계 체인) |
| 인프라 / 배포 | Vercel (정적 호스팅 + 서버리스), GitHub 자동 배포 |
| 관측 | Sentry(`@sentry/node` + 브라우저 로더), Vercel Web Analytics · Speed Insights |
| 보안 | Origin 검사 + HMAC 서명 토큰 + 입력 크기 캡 (전부 무상태) |

## 프로젝트 구조

```
├── public/
│   └── index.html      # 프론트엔드 (UI·대화·과업 카드·완료 보상)
├── api/
│   ├── chat.js         # 대화·과업 생성 (Claude 2단계 체인) + 어뷰징 방어
│   └── token.js        # HMAC 봇 마찰 토큰 발급 + Sentry DSN 전달
├── package.json
└── README.md
```

## 로컬 실행

```bash
# 1) 의존성 설치
npm install

# 2) .env 작성 (아래 환경 변수 참고)

# 3) Vercel CLI로 로컬 구동 (.env 자동 로드)
vercel dev
```

## 환경 변수

| 변수 | 필수 | 설명 |
|---|---|---|
| `ANTHROPIC_API_KEY` | ✅ | Claude API 키 |
| `APP_SECRET` | 선택 | 설정 시 HMAC 봇 마찰 토큰 검증 활성화 (미설정 시 해당 검증은 건너뜀) |
| `SENTRY_DSN` | 선택 | 설정 시 프론트·백엔드 에러 모니터링 활성화 |

> `.env`는 `.gitignore`에 포함되어 커밋되지 않습니다. 프로덕션 값은 Vercel 프로젝트의 Environment Variables에 설정합니다.

## 배포

`main` 브랜치에 push하면 Vercel이 자동으로 프로덕션 배포합니다.

```bash
git push origin main   # → 자동 배포
```
